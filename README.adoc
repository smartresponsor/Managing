= Managing

`Managing` is the EasyAdmin-backed CMS surface mounted at `/manage`.

It discovers business CRUD resources from connected sibling components through Symfony DI providers and renders native EasyAdmin index/detail/new/edit screens.

== Canon

* Component namespace: `App\Managing\...`
* Business prefix: `Manage`
* Public admin mount point: `/manage`
* Primary renderer: EasyAdmin dashboard plus one universal component controller
* Left menu: one entry per business component
* Menu entries open the universal component index controller keyed by the last URL token
* Host application decides which sibling components are listed and which are excluded

== Extension points

* `App\Managing\ServiceInterface\Admin\ManageAdminProviderInterface`
* `App\Managing\ServiceInterface\Admin\ManageAdminRegistryInterface`
* `App\Managing\ServiceInterface\Crud\ManageCrudResourceRegistryInterface`
* `App\Managing\ServiceInterface\Crud\ManageCrudActionUrlBuilderInterface`

Providers are tagged with `manage.admin_provider`.

== Rules

* No separate system screens.
* No legacy route, form, probe, or relation layer.
* No custom HTML shell in PHP.

The dashboard should stay native to EasyAdmin and the menu should point to a single component index controller, not to per-component hardcoded pages.
